GeoSings主控脚本runGeoSings.py介绍

返回主索引

    脚本runGeosings是整个GeoSings产品的主控脚本。你可以用他来控制整个工程。可以用来运行主程序，生成API文档，察看API文档，管理工程，打开编辑源码，或者生成tags文件来在gnu编辑器里进行函数补全等等工作。通过这一个脚本可以完成上面的所有任务。所以管理整个GeoSings是很轻松的一件事。

    启动控制台后，提示行会有这样一行信息：

select a operator as following: {app|doc|mkdoc|mktags|prj|webpub|weblc|initdb|dbshell}
input here :

这是所有的runGeosings脚本可以完成的操作，包括app,doc,mkdoc,mktags,prj,webpub,weblc,initdb,dbshell，下面是这几个命令的解释:

命令名 命令操作行为
app 运行火尖枪主应用程序
doc 察看帮助文档
mkdoc 自动生成API帮助文档
mktags 生成用于vim的补全数据库
prj 运行项目管理程序
wepub 在http://yourip:8000上发布“风火轮”服务
weblc 在http://localhost:8000上发布“风火轮”服务
initdb 初始化“风火轮”后台数据库
dbshell 用python操作“风火轮”后台数据库

一些解释：

  1. mktags操作只对gnu的编辑器如vim，Emacs等来说有用，对于其他编辑器并没有太大用处。把mktags归在runGeoSings脚本中只是让ProjCtrl调用vim使用tags方便。

  2. mkdoc操作可以自动生成api文档。命令生成有几个步骤，
    第一步：

    quiet? (n/y) :

    问的是是否打印生成步骤，默认是打印。

    all? (y/n) :

    问的是要不要生成所有的api文档。默认是全部生成，生成后退出程序，如果选择否，则会分开询问。根据选择生成相应软件包的api文档。
    如果选择了不全部生成，则会有三个询问，分别是

    make core? (y/n) :
    make ui? (y/n) :
    make tools? (y/n) :

    根据选择生成各个所需模块或者不生成指定模块。

  3. webpub和weblc命令是用于调试的。用的是Django的内部服务器。要发布到apache，那还要做很多工作。
